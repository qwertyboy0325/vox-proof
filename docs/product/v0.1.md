# VoxProof v0.1 Product Scope

What it means for v0.1 to be "established", and which claims it may make, is owned by `product/versioning.md`. The frozen v0.1 mechanism gates, qualifying human-evidence rules, mixed zh-EN pre-tag fixture requirement, and release mechanics are owned by MD-007. v0.1 remains in progress and not established.

## Problem Statement

Existing ASR output can fail on mixed Chinese-English speech, specialized terms, names, product names, abbreviations, and localized pronunciations. The user should not need to scan every subtitle segment manually. VoxProof must not silently rewrite content.

## Initial Target Workflow

1. Input an existing SRT and provisional session-scoped terms. Optional audio and future Domain Collections are not current runtime inputs.
2. Detect suspicious spans.
3. Generate bounded candidate corrections.
4. Present evidence and nearby transcript context.
5. Let a human accept, reject, edit, or defer.
6. Materialize a reviewed SRT and decision records.

## v0.1 Pipeline

`Parse / Validate -> Normalize -> Detect CandidateSpan -> Analyze -> Aggregate Evidence -> Rank Review Queue -> Human Review -> Materialize Output -> Export`

## Current Transformation Scope

The implemented v0.1 loop covers only bounded accuracy-oriented term restoration through exact, case-sensitive matching, bounded ASCII-Latin phonetic evidence, and human-accepted replacements. Exact case-sensitive and bounded phonetic behavior are provisional implementation baselines, not universal product rules. The loop does not implement a general correction framework, formatting policy, disfluency cleanup, editorial transformation, Domain Collection, policy resolver, LLM, automation, or multiple-projection system.

The cross-version distinctions among correction categories, policies, transformations, and projections are owned by `product/correction-system-boundaries.md`. They do not add those future capabilities to current v0.1 scope.

## v0.1 Evidence Sources

- Glossary / alias matching.
- Explicit user-supplied or human-confirmed observed-error-form mapping.
- Phonetic or pinyin similarity.
- Mixed-language signal.

The committed bounded ASCII-Latin phonetic producer has exercised both a zero-candidate real-material path and a reference-supported emitted-candidate real-speech path. The committed strict skeleton-compatible `evaluate` mechanism has exercised a deterministic calibration correspondence path on frozen public FLEURS material. These observations support mechanism viability only. MD-007 v0.1 mechanism gates are frozen and decided; product/detector-effectiveness thresholds are not established by the probes and remain outside the v0.1 mechanism claim. Qualifying owner-operated human-review evidence and v0.1 establishment remain pending.

## v0.1 Acceptance Criteria

Mechanism-establishment acceptance is governed by MD-007. The ten frozen mechanism gates in MD-007 D9 must all pass on qualifying human-correction evidence under MD-007 D8. They are not product-effectiveness thresholds.

Operational invariants for the current loop:
- No transcript text changes before user confirmation.
- Every review item retains the original observed text and source location.
- Reviewed output can be derived from the original transcript plus accepted decisions.
- Rules-only operation is possible without any model runtime.
- Output includes reviewed SRT and decision records.
