# VoxProof v0.1 Product Scope

What it means for v0.1 to be "established", and which claims it may make, is owned by `product/versioning.md`.

## Problem Statement

Existing ASR output can fail on mixed Chinese-English speech, specialized terms, names, product names, abbreviations, and localized pronunciations. The user should not need to scan every subtitle segment manually. VoxProof must not silently rewrite content.

## Initial Target Workflow

1. Input an existing SRT and provisional session-scoped terms. Optional audio and future Domain Collections are not current runtime inputs.
2. Detect suspicious spans.
3. Generate bounded candidate corrections.
4. Present evidence and nearby transcript context.
5. Let a human accept, reject, edit, or defer.
6. Materialize a reviewed SRT and decision records.

## v0.1 Pipeline

`Parse / Validate -> Normalize -> Detect CandidateSpan -> Analyze -> Aggregate Evidence -> Rank Review Queue -> Human Review -> Materialize Output -> Export`

## Current Transformation Scope

The implemented v0.1 loop covers only bounded accuracy-oriented term restoration through exact, case-sensitive matching and human-accepted replacements. Exact case-sensitive behavior is a provisional implementation baseline, not a universal product rule. The loop does not implement a general correction framework, formatting policy, disfluency cleanup, editorial transformation, Domain Collection, policy resolver, LLM, automation, or multiple-projection system.

The cross-version distinctions among correction categories, policies, transformations, and projections are owned by `product/correction-system-boundaries.md`. They do not add those future capabilities to current v0.1 scope.

## v0.1 Evidence Sources

- Glossary / alias matching.
- Explicit user-supplied or human-confirmed observed-error-form mapping.
- Phonetic or pinyin similarity.
- Mixed-language signal.

## v0.1 Acceptance Criteria

- No transcript text changes before user confirmation.
- Every review item retains the original observed text and source location.
- Reviewed output can be derived from the original transcript plus accepted decisions.
- Rules-only operation is possible without any model runtime.
- Output includes reviewed SRT and decision records.

## v0.1 Non-Goals

- ASR / Whisper runtime.
- Local LLM runtime.
- Model download management.
- Real-time speech.
- Cloud sync.
- User accounts.
- Collaboration.
- RAG.
- Agents.
- Speaker diarization.
- Full subtitle editing.
- Medical-specific workflows.
- Enterprise integrations.
- Dynamic workflow engines.
- Plugin systems.
