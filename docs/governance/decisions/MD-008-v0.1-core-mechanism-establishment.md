# MD-008 — Establish the bounded v0.1 core mechanism

Status: accepted

Date: 2026-07-18

Decision authority: Ezra explicit approval in the VoxProof project conversation on 2026-07-18

Classification: v0.1 establishment decision

## Context

MD-007 froze the v0.1 establishment boundary, qualifying human-evidence rules, mechanism gates, mixed Traditional-Chinese / ASCII-Latin fixture requirements, and release mechanics.

Qualifying owner-operated real-speech human review under MD-007 D8 passed all ten MD-007 D9 mechanism gates at repository HEAD `7efe8ba`. MD-003 output conformance passed. The authorized mixed Traditional-Chinese / ASCII-Latin fixture required by MD-007 D10 passed at implementation baseline commit `05b7a2f`. Ezra explicitly approved establishment after reviewing the accepted limitations and non-claims.

## Scope

This decision governs only the bounded v0.1 core mechanism defined in MD-007 D7. It does not establish product effectiveness, external-user validation, or release/tag activity by itself.

## Decision

Establish VoxProof v0.1 as providing a bounded, deterministic post-ASR transcript-review mechanism with:

- strict and traceable SRT parsing and source anchors;
- bounded evidence generation and `ReviewCase` construction;
- explicit human correction authority;
- reviewed output derived from immutable source content plus accepted applicable decisions;
- deterministic decision log and session summary;
- strict skeleton-compatible `compare` and `evaluate` correspondence support;
- fail-closed refusal for incompatible inputs, invalid anchors, overlaps, or other materialization mismatches.

Implementation baseline:

```text

```

The annotated `v0.1.0` tag must target the later commit containing this accepted decision and the canonical establishment status updates, not the implementation baseline alone.

## Accepted limitations

- Owner-operated and non-blind human review.
- Raw ASR text was visible; raw-ASR anchoring was possible.
- Prior community/reference exposure existed.
- Reference transcript was not correctness authority.
- Reviewer English/accent/domain familiarity was limited.
- MD-007 D9 gate 6 passed vacuously with zero rejected decisions (`0/0`).
- Temporary study-local review GUI was not product GUI.
- Missing artifact `calibration-comparison-v0.patch` remains lost / not exactly recoverable:
  - historical size: 42536 bytes
  - historical SHA-256: `67552483588a464219f992b6492e94c7a006a77a6b2981c8089f220a6e9cda4e`
- Current implementation and qualifying evidence do not depend on that missing patch.

## Assumptions / unresolved questions

- The implementation baseline at `05b7a2f` remains the measured mechanism baseline even though the tag target is a later documentation commit.
- External facilitated/target-cohort validation remains deferred to v0.2.
- Research ZIPs, seals, and attestations remain local untracked artifacts unless separately authorized.

## Alternatives considered

- Defer establishment until external-user validation: rejected; external validation belongs to v0.2, not v0.1 mechanism establishment under MD-007.
- Treat the mixed Traditional-Chinese / ASCII-Latin fixture as real-speech or product evidence: rejected; D10 remains synthetic mechanism evidence only.
- Tag the implementation baseline directly without the MD-containing commit: rejected; tag target must include this accepted decision and canonical status updates.
- Waive or reinterpret D9 gate 6 because no rejected alternatives occurred: rejected; gate 6 passed vacuously with explicit `0/0` evidence.
